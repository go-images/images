**A pure-Go (no cgo) image-processing library** in the style of
[scikit-image](https://scikit-image.org/), built entirely on the Go standard
library's `image`, `image/color`, `image/png` and `image/jpeg` packages.

Every operation is a pure function: it takes an image and returns a freshly
allocated `*image.RGBA`, never mutating its input.

## Why not just use a C library?

The established image stacks all need a native C dependency: ruby-vips wraps
**libvips**, RMagick wraps **ImageMagick**, and the cgo-backed Go wrappers pull
in the same shared libraries. That makes cross-compilation and embedding
awkward. `go-images/images` is **CGO=0**: it builds and runs identically on
every Go target with no system packages, which makes it trivially
cross-compilable and embeddable (for example inside
[go-embedded-ruby](https://github.com/go-embedded-ruby/ruby)).

Pure-Go image libraries already exist
([bild](https://github.com/anthonynsimon/bild),
[disintegration/imaging](https://github.com/disintegration/imaging)); the goal
here is a clean, correct, fully test-covered core whose pixel kernels are
isolated so they can later be swapped for SIMD implementations generated by
[go-asmgen](https://github.com/go-asmgen) across the six supported 64-bit
targets (amd64, arm64, riscv64, loong64, ppc64le, s390x). See
[docs/plan-images.md](docs/plan-images.md) for the roadmap and an honest
comparison.

## API

```go
import "github.com/go-images/images"
```

Conversion and I/O:

- `images.ToRGBA(img image.Image) *image.RGBA`
- `images.Load(path string) (*image.RGBA, error)`
- `images.Save(path string, img image.Image) error` ŌĆö format by extension (`.png`, `.jpg`, `.jpeg`)
- `images.Decode(r io.Reader) (*image.RGBA, error)` ŌĆö format auto-detected
- `images.Encode(w io.Writer, img image.Image, format images.Format) error` ŌĆö `images.PNG`, `images.JPEG`

Operations (each returns a new `*image.RGBA`):

Point & colour:

- `images.Grayscale(img)` ŌĆö luminance-weighted (Rec. 601)
- `images.Invert(img)`
- `images.AdjustBrightness(img, delta)` ŌĆö clamped to `[0, 255]`
- `images.AdjustContrast(img, factor)` ŌĆö about mid-point 128, clamped
- `images.RGBToHSV(img)` / `images.HSVToRGB(img)` ŌĆö byte-encoded, round-trip-stable
- `images.OtsuThreshold(img)` ŌĆö Otsu level (matches `skimage.filters.threshold_otsu`)
- `images.Threshold(img, t)` / `images.Otsu(img)` ŌĆö binarise on luminance

Filters:

- `images.Convolve(img, images.Kernel{...})` ŌĆö arbitrary odd kernel, clamp-to-edge
- `images.GaussianBlur(img, sigma)` ŌĆö separable Gaussian
- `images.BoxBlur(img, radius)` ŌĆö separable running-sum mean (matches `scipy.ndimage.uniform_filter`)
- `images.Sobel(img)` ŌĆö gradient-magnitude edge map (on luminance)
- `images.SobelX(img)` / `images.SobelY(img)` ŌĆö directional Sobel responses (mid-grey = zero gradient)

Morphology (grayscale square element, also binary on 0/255 images):

- `images.Erode(img, r)` / `images.Dilate(img, r)` ŌĆö local min / max
- `images.Open(img, r)` / `images.Close(img, r)` ŌĆö erodeŌåÆdilate / dilateŌåÆerode

Geometry:

- `images.Resize(img, w, h, mode)` ŌĆö `images.NearestNeighbor` or `images.Bilinear`
- `images.FlipHorizontal(img)` / `images.FlipVertical(img)` ŌĆö `numpy.fliplr` / `flipud`
- `images.Rotate90(img)` / `images.Rotate180(img)` / `images.Rotate270(img)` ŌĆö `numpy.rot90`
- `images.Crop(img, image.Rect(x0, y0, x1, y1))`

### Example

```go
src, err := images.Load("in.png")
if err != nil {
    log.Fatal(err)
}
gray := images.Grayscale(src)
blurred, err := images.GaussianBlur(gray, 2.0)
if err != nil {
    log.Fatal(err)
}
if err := images.Save("out.png", blurred); err != nil {
    log.Fatal(err)
}
```

## Performance

On a 512├Ś512 image (arm64 Linux, same VM for both stacks), the scalar pure-Go
kernels already **beat scikit-image / SciPy on the separable hot ops** ŌĆö box
blur **3.1├Ś** and Sobel **3.2├Ś** faster ŌĆö while the arithmetic-heavy Gaussian
and morphology kernels are ~1.1├Ś behind NumPy's compiled C (the targets for the
planned go-asmgen SIMD + multicore work). Box blur and grayscale morphology
match SciPy bit-for-bit; Gaussian matches within one LSB. See
[docs/perf.md](docs/perf.md) for the full method, tables and correctness checks.
